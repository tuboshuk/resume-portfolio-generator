# Iteration 1 Candidates

本文件用于汇总下一迭代候选需求，默认按“提升成交/降低交付成本/提高演示效果”的优先级排序。

## Iteration 1 推荐 Top3（默认）

1. 一键发布（GitHub Pages）
2. 配置校验与友好报错（内容模型守护）
3. 示例模板与交付包（买家 5 分钟跑起来）

## P0（强烈建议）

1. 内容校验与友好报错
   - 当 `site.json` / `projects.json` 缺字段或类型不对时，能在本地开发时给出清晰提示。
   - 验收标准：错误信息包含字段路径、期望类型、示例修复方式；构建阶段直接失败（不生成错误站点）。
2. 一键发布到 GitHub Pages 的工作流
   - 增加 `GitHub Actions` 工作流，自动构建 `out/` 并发布。
   - 验收标准：main 分支 push 后自动发布；自带一个可复制的配置说明。
3. 生成演示数据与导出模板
   - 提供 `site.example.json`、`projects.example.json` 与复制说明。
   - 验收标准：README 中给出 3 步替换流程；示例数据能完整覆盖首页/列表/详情页所有模块。

4. 交付清单与演示脚本（30-60 秒）
   - 提供一份“如何演示/如何交付/如何部署”的固定脚本与截图要点。
   - 验收标准：新用户照着脚本能在 1 分钟内讲清楚价值并完成一次构建。

## P1（可选增强）

1. SEO 与 OpenGraph 完善
   - 首页/列表/详情的 title/description/og:image。
   - 验收标准：每个项目详情页有独立的 title/description；分享链接展示正确的封面图。
2. 主题切换（浅色/深色）
   - 保持内容结构不变，只换视觉。
   - 验收标准：不影响静态导出；默认跟随系统主题（可选）。
3. 图片加载与性能
   - 静态导出下的图片策略优化（本地图片/远程图片/占位图）。
   - 验收标准：首屏 LCP 改善；图片加载不抖动（有尺寸占位）。

4. 简历下载与文件托管
   - 支持把 PDF 放到 `public/` 并在首页一键下载。
   - 验收标准：README 给出放置路径；站点内有明显入口。

5. 导航与信息架构增强
   - 首页增加“回到顶部”、目录高亮（可选）、更清晰的内容层级。
   - 验收标准：移动端也可用；无明显布局抖动。

## P2（后续再说）

1. MDX/Markdown 写项目详情
2. 多语言（中英切换）
3. 表单化配置生成器（Web UI）

4. CLI 初始化工具
   - `npx create-xxx` 一键生成站点模板与示例数据。

5. Blog / Notes
   - 增加文章列表与详情（用于内容营销与 SEO）。

## 角色分工建议（用于开迭代时拆任务）

- PM：明确 Top3 的验收标准与演示脚本。
- Tech Lead：确定发布方案与配置校验策略。
- FE：实现页面与交互、主题/SEO。
- QA：补充端到端验收清单，覆盖错误/空状态与部署验证。
